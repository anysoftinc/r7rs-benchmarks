Scheme Benchmarks

Based on the Larceny benchmarks. Code on GitHub

Tests were run on an Intel(R) Core(TM) i3-N305 CPU @ 3.8GHz with 48GB of RAM by using the Arch Linux packages. No guarantees for any of the numbers. CSV raw data

Generated at 2024-07-06, e-mail

SAFE vs. UNSAFE optimizations

The benchmark results on this page are collected in safe mode. This means various optimisations are turned off for each implementation. If you need code to run as fast as possible, look into that, the Readme on GitHub contains a few pointers for the different implementations.

Feel free to run the benchmarks yourself without safety turned on. You'll probably need to modify the bench file for that.

Actual R7RS Support

Many of these implementations do not fully implement R7RS, but instead there is a bit of "shim" code. You can find this by looking at src/<Name>-prelude.scm and src/<Name>-postlude.scm, to see which changes are necessary. Some changes are also made by the bench script, especially relating to imports.

However, all of these implementations are "close enough" to R7RS to run a non-trivial amount of code.

Missing implementations

Some implementations are missing. This is a short explanation on why:
  • Owl-Lisp: dialect too far from Scheme to run the benchmarks, doesn't have e.g. begin or library (scheme read)
  • RScheme: cannot build.
  • Rhizome/Pi: cannot build.
  • TinyLisp: did not finish any of the benchmarks within 5 minutes.
